html : 웹문서의 포맷(화면에 보이는 모양)을 꾸며주기 위해 존재하는 Markup Language

css : html의 꾸며주기 기능의 단조로움을 보완하기 위해 만들어진 디자인용 언어

Javascript : 웹 클라이언트 측에 상호작용이 필요한 기능을 만들기 위해 지원되는 프로그래밍 언어

xml : 원래는 html의 대체 용도로 만들어졌지만
	지금은 임의의 포맷을 사용자가 정의할 수 있는 도구로 발전되었다

json : Java Script Object Notation
	원래는 Javascrpit언어에서 오브젝트를 나타내는 문법 용도였는데
	지금은 임의의 포맷을 사용자가 정의할 수 있는 도구로 발전되었다


UI Toolkit

	html, xml	->	uxml
	css			->	uss		<-- uss를 바꾸면 디자인을 한방에 바꿀 수 있다
	javascript	->	C#

	uxml, uss : 사람이 눈으로 보고 편집 가능

	데이터 드리븐 data driven 방식이 적용되어 있다
	: 게임에 필요한 데이터들을 사전에 정의해두고 '파일' 형태로 만들어두는 것
		<-- 작업분담의 단위로 '파일'을 사용하기 위해서다


파일 형태로 저장, 로드 하는 형식은 다음 두 가지가 있다

	text vs binary

	text
		장점
			사람의 눈으로 보고 편집 가능

		단점
			같은 데이터라면 일반적으로 크기가 커짐

	binary
		장점
			같은 데이터라면 일반적으로 크기가 작음

		단점
